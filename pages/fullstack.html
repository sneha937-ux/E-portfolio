Section 2 · Definition

What is Full Stack Development?

Full stack development refers to the practice of designing, building, and maintaining both the client-facing (frontend) and server-side (backend) layers of a web application, along with the database layer. A developer who works across all of these layers is referred to as a full stack developer (Mead 2019, p. 12).

The term 'stack' refers to the collection of technologies used to build the application, typically grouped into three tiers: the presentation layer (what the user sees), the logic layer (how the application behaves), and the data layer (where data is stored). According to Duckett (2014, p. 6), the separation of these concerns is a foundational principle of modern software architecture.

Section 2.3 · Communication

How the Layers Communicate

The three layers communicate through a defined request-response cycle. When a user interacts with the frontend — for example, submitting a login form — the browser sends an HTTP request to the backend server. The server processes the request, queries the database if needed, and returns an HTTP response (usually JSON data) back to the frontend (Mozilla Developer Network 2024).

Example HTTP Request Cycle:

  1. User clicks 'Login' on the frontend form
  2. Browser sends POST /api/login HTTP Request with credentials
  3. Backend validates credentials against the database
  4. Database returns the user record to the backend
  5. Backend signs a JWT token and sends 200 OK HTTP Response
  6. Frontend stores the token and redirects to the dashboard

Frontend Code — fetch() API

login.jsJavaScript
// Frontend: Send a login request to the backend API
async function loginUser(email, password) {
  const response = await fetch('/api/login', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify({ email, password })
  });
  const data = await response.json();

  if (data.success) {
    localStorage.setItem('token', data.token);
    window.location.href = '/dashboard';
  } else {
    alert('Login failed: ' + data.message);
  }
}

Backend Code — Node.js / Express

server.jsNode.js
const express = require('express');
const bcrypt  = require('bcrypt');
const jwt     = require('jsonwebtoken');
const app     = express();

app.post('/api/login', async (req, res) => {
  const { email, password } = req.body;
  const user = await User.findOne({ email });

  if (!user || !await bcrypt.compare(password, user.hash)) {
    return res.status(401).json({ success: false, message: 'Invalid credentials' });
  }

  const token = jwt.sign({ id: user.id }, process.env.JWT_SECRET, { expiresIn: '24h' });
  res.json({ success: true, token });
});

Source: Auth0 (2023); Flanagan (2020, p. 375)

Section 2.4–2.5 · Stacks

Advantages & Common Technology Stacks

✔ Advantages

  • Work independently across all layers
  • Better understanding of how components interact
  • Faster end-to-end debugging
  • Highly employable in startups and small teams
  • Reduced need for separate frontend/backend developers

✘ Challenges

  • Breadth of knowledge required is very wide
  • Keeping up with rapid changes in each layer
  • Risk of being a 'jack of all trades, master of none'
  • Difficult to specialise deeply in one area
  • Full stack expertise takes significant time to develop

Common Technology Stacks (Wieruch 2020, p. 8):

  • MERN: MongoDB, Express.js, React.js, Node.js — JavaScript full stack
  • LAMP: Linux, Apache, MySQL, PHP — classic server-side stack
  • MEAN: MongoDB, Express.js, Angular, Node.js
  • Django + React: Python backend with React frontend
